# Lead Capture Flow

Use this as the default lead capture structure for Micah client installs.

## Required Fields

1. Caller name
2. Mobile number
3. Email address
4. Reason for enquiry
5. Best callback time

## Optional Fields

- Business name
- Suburb or service area
- Preferred service
- Urgency
- Existing customer status
- Booking preference

## Capture Rules

- Capture one detail at a time.
- Confirm details naturally using "is that right?"
- Do not ask again for a detail that has already been captured and confirmed.
- If the caller skips a detail, continue with the next useful detail and return to the missing item later.
- If the caller is distressed, urgent, or unwell, prioritise reassurance and escalation.

## Handoff Trigger

Send the lead summary once enough caller details are captured for the client to act on the enquiry, or when the call is ending.

## Default Completion Line

> Thanks, [Name]. I've got that down and I'll pass it through so someone can follow up with you.
